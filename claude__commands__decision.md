Запрос: $ARGUMENTS

## Инструкции

1. Если `$ARGUMENTS` пустой или равен `all` — прочитать `docs/00_DECISIONS_LOG.md` целиком и вывести summary всех решений (одна строка на каждое).

2. Если `$ARGUMENTS` соответствует ID (`C1`, `C12`, `Q3`, `F1` и т.п.):
   - Прочитать `docs/00_DECISIONS_LOG.md`.
   - Извлечь именно этот пункт + все, которые его явно упоминают.
   - Если решение ссылается на другой документ (`docs/XX_*.md`) — прочитать релевантный раздел и привести.
   - Вывести: ID, формулировку, обоснование, последствия, связанные решения.

3. Если `$ARGUMENTS` — keyword (например `cooldown`, `compression`, `chat_id`):
   - `rg -i "<keyword>" /var/www/oi-tracker/docs/00_DECISIONS_LOG.md` чтобы найти упоминания.
   - Вывести все совпавшие решения с контекстом.

4. **Никогда** не «угадывай» содержимое решений — только цитируй из файла.

5. Если запрошенного ID нет в файле — явно сказать «решение Xn не найдено в 00_DECISIONS_LOG.md» и предложить:
   - либо это новый Fxx (предложить шаблон записи),
   - либо опечатка (показать список ID, которые есть).

## Шаблон новой записи Fxx

Если пользователь хочет добавить новое решение, предложить:

```markdown
### Fn — <короткая формулировка>
**Дата:** YYYY-MM-DD
**Контекст:** что произошло, какое решение из C/Q пересматриваем (если применимо).
**Решение:** что теперь делаем.
**Обоснование:** почему изменение оправдано.
**Последствия:** что меняется в коде / docs / процессе.
**Связано с:** Cxx, Qxx, Fxx (если применимо).
```
